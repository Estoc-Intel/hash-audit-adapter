# Security policy

This repository intentionally exposes only a fixed dictionary audit. Changes that add arbitrary Hashcat arguments, shells, masks, rules, interactive controls, persistent state, credential injection, or runtime network access are out of scope and should be rejected.

Do not include real hashes, passwords, wordlists, accounts, targets, or task outputs in issues, tests, commits, or pull requests. Tests must use synthetic fixtures only.

Report security concerns privately to the repository owner instead of posting sensitive details in a public issue.
